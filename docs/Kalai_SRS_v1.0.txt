1. Introduction
1.1 Purpose
This Software Requirements Specification defines the complete technical and functional requirements for Kalai — an autonomous AI marketing agent for Indian SMBs. This document serves as the authoritative reference for development, QA, and stakeholder alignment. It covers all user-facing features, system behaviours, API integrations, data models, performance benchmarks, and security requirements.

1.2 Scope
Kalai is a multi-platform SaaS application comprising:
Web Application: Next.js 14 App Router (primary product surface, mobile-responsive PWA)
Mobile Application: React Native + Expo (iOS and Android, ships V2)
Shared Packages: Turborepo monorepo with @kalai/types, @kalai/api-client, @kalai/validators, @kalai/utils, @kalai/ai-prompts
Backend: Supabase (PostgreSQL + pgvector + Auth + Storage), Inngest (job queue), Vercel (hosting), Cloudflare R2 (media storage)
AI Layer: Claude Sonnet 4.6 (copy), Nano Banana Pro / Gemini 3 Image API (visuals), Gemini Flash (strategy/analysis)
Platform Integrations: Meta Graph API v21, LinkedIn API v2, Google Business Profile API, YouTube Data API v3, Razorpay

2. System Architecture Overview
2.1 High-Level Architecture
Kalai follows a serverless-first architecture optimised for rapid development, low operational overhead, and automatic scalability. All components are managed services — there are no self-managed servers in the V1–V3 architecture.

5.2.3 Tamil/Hindi Text Rendering
Nano Banana Pro is the only image generation model that reliably renders Tamil and Hindi text inside generated images. This is a critical differentiator for Kalai's Indian market positioning. When generating posts with text in Tamil or Hindi:
Specify language explicitly in the prompt: 'text in Tamil script (Unicode)'
Specify font style: 'clear, legible, bold Tamil typography'
Validate output: Post-generation OCR check via Gemini Vision to verify text accuracy
Fallback: If Tamil/Hindi text fails validation, generate English version + manual review flag

5.4 pgvector Brand Brain — Vector Storage
5.4.1 Embedding Strategy
Embedding model: text-embedding-3-small (OpenAI, 1536 dimensions)
What gets embedded: Onboarding answers, past post captions (top performers), project descriptions, brand voice samples
Query at post generation: SELECT content FROM brand_voice_embeddings WHERE brand_id = $1 ORDER BY embedding <=> $query_embedding LIMIT 5
Result injection: Top 5 semantically similar voice samples injected into Claude system prompt
Update frequency: New embeddings added after each approved post, weighted by performance

7. Inngest Job Queue — Background Tasks
7.1 Job Definitions

7.2 Error Handling Strategy
All jobs wrapped in try/catch. Errors logged to Sentry with full context.
publishPost: 3 retries with exponential backoff (1min, 5min, 15min). After 3rd failure, post status = 'failed', Sentry alert + email to user.
generateCalendar: If AI generation fails mid-calendar, save completed posts, resume from failure point.
processWebhookDM: Webhook acknowledgement (200 OK) sent immediately, processing happens asynchronously. Never block Meta webhooks.
sendSundayDigest: If email fails, retry once. If user has no engagement data, send simplified version.
